LifeCycle Methods

Overview

Slice.js components expose three lifecycle methods for predictable, explicit behavior:

  • init() for one-time setup
  • update() for refreshes when data or routes change
  • beforeDestroy() for cleanup and memory safety

These methods are called by the framework, and are the recommended places to manage component state, subscriptions, and DOM updates.

When Each Method Runs

Navigation and Reuse

update() is called when a cached component is reused by routing (e.g., MultiRoute), which keeps UI responsive without rebuilding static structure.
