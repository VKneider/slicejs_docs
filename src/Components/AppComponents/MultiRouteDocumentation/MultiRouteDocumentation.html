Slice MultiRoute

To build a simple Slice MultiRoute you only need to use the "slice.build" method with a routes array prop. The MultiRoute component automatically applies default values and provides enhanced debugging capabilities through Static Props. It handles dynamic component rendering based on the current URL pathname with built-in component caching for optimal performance.

Advanced Usage

You can configure multiple routes with different components. The MultiRoute component automatically matches the current pathname with your defined routes and renders the corresponding component. Static Props ensure that the routes property is properly typed as an array and provides validation in development mode.

Static Props Integration

The MultiRoute component leverages Static Props for improved developer experience:

  • Automatic defaults: All props have sensible defaults applied automatically
  • Type validation: Development-time warnings for incorrect prop types
  • Enhanced debugging: Right-click any MultiRoute to see detailed prop information
  • Self-documenting: Props are clearly defined with types and defaults
  • Component caching: Built-in performance optimization with component reuse

MultiRoute Static Props Configuration:

static props = {
   routes: { type: 'array', default: [], required: true }
};
