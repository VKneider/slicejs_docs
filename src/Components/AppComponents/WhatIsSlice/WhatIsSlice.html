Slice Manifesto

Slice is born to teach by building.

Slice.js is born to be a framework where you can learn, in a practical way, the core patterns behind component-based programming and modern frameworks.

Its purpose is to help you build complete web applications from a vanilla perspective, without non-standard abstractions or syntax outside HTML, CSS, and JavaScript. It does not seek to compete. It seeks a different way of building.

  • Web Components
  • Learning by doing
  • Architecture with intent

A practical ecosystem for building and understanding modern frontend systems.

Why

To close the gap between documentation theory and the daily reality of building products.

How

By embracing native browser APIs, explicit component lifecycles, and a transparent architecture.

What it is not

A trend chase. Slice is a long-term learning and shipping framework, not a popularity race.

Ecosystem

Ecosystem Projects

Slice grows as a connected ecosystem: framework, documentation, visual resources, and MCP tooling.
