What is Slice.js?

Slice.js is a vanilla, lightweight framework for building functional frontend applications while learning the real concepts behind modern frameworks. You work directly with Web Components, plain JavaScript, and modern browser APIs.

It is intentionally small, practical, and transparent: you can see how routing, state, services, and UI composition actually work without a heavy abstraction layer.

Core Principles

Vanilla-First

Learning-Driven

Small Surface Area

Practical Architecture

Key Features

  • slice.build: Build visual and service components with a single API
  • Static Props: Explicit public props with defaults and validation
  • Routing: Route and MultiRoute containers with caching + update()
  • ContextManager: Shared state with watchers, selectors, and persistence
  • EventManager: Lightweight pub/sub for app signals
  • Services: Fetch, storage, and reusable business logic
  • Themes: CSS-variable driven themes via StylesManager
  • CLI: init, dev, bundle, and component management

Slice.js Architecture

Slice.js uses a three-layered architecture that keeps UI, structure, and logic independent while still working together:

Visual Components

UI Elements

Structural Components

Core Framework

Business Logic

Visual Components

Render UI, handle interactions, and manage view state. Examples include buttons, inputs, cards, and layouts.

Button.js, Card.js, Input.js

Structural Components

Coordinate the app: routing, component lifecycle, themes, events, and shared state.

Router.js, Controller.js, StylesManager.js, ContextManager.js, EventManager.js

Service Components

Encapsulate data access and business logic. Services can cache results and integrate with context or events.

FetchManager.js, IndexedDbManager.js, LocalStorageManager.js

Data Flow

Data flow is explicit and simple:

  1. User interacts with Visual Components
  2. Visual Components call Services for data or logic
  3. Services emit events or update Context
  4. Visual Components refresh via update() or watchers
  5. Structural components coordinate routing and lifecycle

Component Types

Visual Components

Visual components render UI and extend HTMLElement. Use Static Props to define a clear public API.

export default class Button extends HTMLElement {
  static props = {
    value: { type: 'string', default: 'Button' },
    onClickCallback: { type: 'function', default: null }
  };

  constructor(props) {
    super();
    slice.attachTemplate(this);
    this.$value = this.querySelector('.slice_button_value');
    slice.controller.setComponentProps(this, props);
  }

  async init() {
    this.$value.textContent = this.value;
    this.addEventListener('click', () => this.onClickCallback?.());
  }
}

Service Components

Services encapsulate data access and logic. Build them once and reuse across routes.

const dataService = await slice.build('FetchManager', {
  baseUrl: 'https://api.myapp.com/v1',
  sliceId: 'data-service'
});

export default class ProductList extends HTMLElement {
  async init() {
    this.dataService = slice.getComponent('data-service');
    await this.load();
  }

  async load() {
    await this.dataService.request('GET', null, '/products', (data) => {
      this.products = data;
      this.update();
    });
  }
}

Component Lifecycle

Use lifecycle methods to separate one-time setup from updates:

  • constructor: Attach template, cache DOM, set props
  • init: One-time setup and async work
  • update: Refresh data and rebuild dynamic lists
  • beforeDestroy: Cleanup timers, listeners, and subscriptions

Shared State

Use ContextManager to share state across components with efficient updates.

slice.context.create('auth', { isLoggedIn: false });

export default class AccountMenu extends HTMLElement {
  async init() {
    slice.context.watch(
      'auth',
      this,
      (isLoggedIn) => this.classList.toggle('signed-in', isLoggedIn),
      (state) => state.isLoggedIn
    );
  }
}

Build a Small App

Create a Component

A minimal component with a counter and update logic:

Counter.js

export default class Counter extends HTMLElement {
  static props = { count: { type: 'number', default: 0 } };

  constructor(props) {
    super();
    slice.attachTemplate(this);
    this.$count = this.querySelector('.count');
    this.$increment = this.querySelector('.increment');
    this.$decrement = this.querySelector('.decrement');
    slice.controller.setComponentProps(this, props);
  }

  async init() {
    this.$increment.addEventListener('click', () => this.increment());
    this.$decrement.addEventListener('click', () => this.decrement());
    this.update();
  }

  async update() {
    this.$count.textContent = this.count;
  }

  increment() {
    this.count += 1;
    this.update();
  }

  decrement() {
    this.count -= 1;
    this.update();
  }
}

Use the Component

Build an instance and attach it to a container:

const counter = await slice.build('Counter', { count: 5 });

const container = this.querySelector('.my-section');
container.appendChild(counter);

console.log(counter.count); // 5
counter.count = 10;
counter.update();
