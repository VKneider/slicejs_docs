Quick API Reference

The public slice.* surface in one page. Each section links to its full guide. Method names use arg? for optional arguments and → for the return.

Components & props

See [Component Anatomy](/Documentation/Component-Anatomy) and [The build method](/Documentation/The-build-method).

Reserved props (stripped before your setters run): id (HTML id), sliceId (registry id, auto-generated if omitted), singleton (Service-only get-or-create).

Cleanup

Destroying runs beforeDestroy() and cleans the registry — see [Common Gotchas](/Documentation/Common-Gotchas).

Router

Route params use ${param} (not :param); each captures one segment and arrives as a string. See [Routing](/Documentation/Routing) and [Route Guards](/Documentation/Routing/Guards).

Context (shared state)

See [ContextManager](/Documentation/Structural/ContextManager) and [Context vs Events](/Documentation/Structural/Context-vs-Events).

Events (pub/sub)

See [EventManager](/Documentation/Structural/EventManager).

Logger

The old logError / logWarning / logInfo names still work but are deprecated — prefer error / warn / info. See [Logger](/Documentation/Structural/Logger).

Environment & theme

See [Environment Variables](/Documentation/Configuration/environment-variables) and [Themes](/Documentation/Themes).
