The Build Method

The slice.build() method is the primary way to create components in Slice.js. It's an asynchronous function that instantiates, configures, and initializes components, automatically handling resource loading, templates, and styles.

Syntax

Parameters

componentName string

The name of the component to build, as registered in components.js

props Object (optional)

An object containing properties to pass to the component. These properties will be available in the component's constructor.

Return Value

Returns a Promise that resolves to the built component instance ready to use. If construction fails, returns null.

Using sliceId

The sliceId is a unique identifier that allows you to retrieve the component later using slice.controller.getComponent(). It's very useful when you need to access the component from other parts of your application.

Build Lifecycle

When you call slice.build(), the following steps occur in order:

  1. Verify that the component exists in the component registry
  2. Load the component's HTML template (if not cached)
  3. Load the component's JavaScript class (if not cached)
  4. Load the component's CSS styles (if not cached)
  5. Create an instance of the component with the provided properties
  6. Apply Static Props if defined
  7. Call the component's init() method if it exists
  8. Register the component in the Controller
  9. Return the ready-to-use component instance

Best Practices

  • Always use await: slice.build() is asynchronous, always use await
  • Provide unique sliceId: For components you need to reference later
  • Handle errors: Use try-catch to handle construction failures
  • Use init() for logic: Place initialization logic in init(), not in the constructor
  • Don't manipulate DOM in constructor: The component's DOM isn't ready until after the constructor
