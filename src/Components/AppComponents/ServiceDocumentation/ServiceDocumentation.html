Services

Overview

Services centralize logic and data access. They can cache results, encapsulate API calls, and integrate with EventManager or ContextManager.

Services are created with slice.build() like other components, but they do not render UI.

Built-in Services

Using a Service

Service Singleton Pattern (sliceId)

Use a stable sliceId to create a shared service instance and retrieve it from any component.

Use sliceId when you need a single instance across routes or screens.
Do not store service instances or functions in slice.context. Persist only serializable state.

Where services live and how to share them

Most services are app-lifetime singletons. Where to build them, how to share one instance, the singleton: true get-or-create shortcut, Provider Services for global UI, and the cleanup contract all live in one canonical place:
