Slice.js CLI Commands

Slice.js includes a powerful command-line interface (CLI) distributed as a binary executable (slice) that helps you create, manage, and run your applications. The CLI is organized into logical command groups for better workflow management.

üöÄ Project Lifecycle Commands

Essential commands to initialize and run your Slice.js application

slice init

Essential

Initializes a new Slice.js project with the complete folder structure, installs initial Visual components, and configures npm scripts.

slice dev / start

Development

Starts the development server for your Slice.js application. Serves your application from the /src directory with live reload support.

üîß Local Component Management

Create and manage custom components specific to your project

slice component create

Creation

Creates new local components with the proper structure. Generates JavaScript, HTML, and CSS files for your component interactively.

slice component list / slice list

Query

Shows all available components in your local project organized by category. Also available as a top-level shortcut: slice list

slice component delete

Deletion

Safely removes components from your local project, ensuring that all related files are properly deleted.

üìö Component Registry Management

Access and install components from the official Slice.js repository

slice get / slice registry get

Installation

Downloads and installs components from the official Slice.js repository. You can install multiple components at once and specify options like force overwrite or service components. Also available via slice registry get.

slice browse / slice registry list

Browse

Shows all available components in the official repository, helping you discover new components and understand what's available for installation. Also available via slice registry list.

slice sync / slice registry sync

Update

Updates all your local components to the latest versions from the repository, ensuring you have the most recent bug fixes and improvements. Also available via slice registry sync.

‚öôÔ∏è Maintenance Commands

Keep your CLI and framework up to date and diagnose issues

slice version

Info

Displays the current version of the Slice.js CLI.

slice update

Updates

Checks for and installs available updates for both the CLI and the Slice.js framework. Supports automatic updates and selective package updates.

slice doctor

Diagnostics

Runs comprehensive diagnostics to check project health, verifying folder structure, dependencies, and configuration.

‚ùì Help Commands

Get help and information about available commands

slice --help / slice help

Help

Displays comprehensive help information about all available commands, options, and usage examples.

üí° Pro Tips

üìã Command Groups

The CLI is organized into logical groups for better workflow management:

Project Lifecycle: init, dev, start
Local Components: component create, component list, component delete
Registry Operations: registry get, registry list, registry sync
Quick Shortcuts: get, browse, sync, list
Maintenance: version, update, doctor

üñ•Ô∏è Multiple Terminals

For the best development experience, keep one terminal running slice dev and use another terminal for other commands.

üè∑Ô∏è Component Categories

When creating components, choose the appropriate category:

  • Visual: For UI components that users interact with (requires HTML, CSS, JS)
  • Service: For components that provide functionality without UI (JS only)
  • AppComponents: For application-specific components (JS only)

üîÑ Registry vs Local

Use slice get for official components and slice component create for custom components specific to your project.
